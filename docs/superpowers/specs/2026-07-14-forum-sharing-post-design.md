# 服务端交流论坛分享帖设计

## 目标

以不熟悉 Linux、Docker 和源码编译的普通玩家视角，分享使用本项目搭建 AzerothCore + Playerbots 服务端的原因、过程和结果，让同类用户能快速判断项目是否适合自己，并找到可直接执行的安装命令。

## 受众

- 服务端交流论坛中的魔兽世界私服爱好者。
- 想搭建自用服务器，但缺少 Docker 和编译经验的用户。
- 关注 Playerbots、内存要求、模块集成和国内网络下载问题的用户。

## 表达方式

- 使用第一人称和普通玩家口吻，避免项目公告式语言。
- 说明自己不懂复杂技术，只是按照项目文档准备服务器并执行命令。
- 保留必要术语，但紧接通俗解释。
- 不虚构安装耗时、性能数据、故障经历或未经验证的兼容性。
- 不夸大“一键安装”，明确 Docker、root 权限、系统版本和内存前提。

## 内容结构

1. 标题突出“不懂技术也能搭建”和 Playerbots。
2. 开头说明搭建动机，以及过去被编译、数据库和 Docker 配置劝退。
3. 用普通用户能理解的方式介绍项目自动完成的工作。
4. 列出支持的 Linux 系统和 4GB、8GB、12GB～16GB 内存建议。
5. 给出官方一键安装命令和一个国内加速命令。
6. 概括 12 个默认模块，重点说明 Playerbots、自动平衡、范围拾取、幻化和拍卖行机器人。
7. 给出安装后的创建账号、连接客户端和常用管理命令。
8. 提醒安装目录、第三方代理、构建耗时和低内存风险。
9. 结尾提供 GitHub 项目地址，并以交流使用体验收尾。

## 事实来源

帖子中的安装命令、系统支持、内存建议、模块列表和管理命令均以项目当前 `README.md`、`ac.conf` 和安装脚本为准。

## 验收标准

- 读起来像普通使用者分享，而不是开发者发布公告。
- 不要求读者理解 Docker Compose、Buildx 或源码构建原理。
- 安装命令可以直接复制，项目地址清晰可见。
- 关键前提和风险没有被宣传性语言掩盖。
- 默认集成的 12 个模块无遗漏、无额外虚构。
